# destinations de la capitalisation

Chaque capitalisation a une destination precise. Pas de "noter pour plus tard".

| Type | Destination | Action |
|------|-------------|--------|
| Anti-pattern | `docs/conventions/anti-patterns.md` | Ajouter avec description + raison + exemple |
| Bon pattern backend | `docs/conventions/backend.md` | Ajouter avec exemple de code |
| Bon pattern frontend | `docs/conventions/frontend.md` | Ajouter avec exemple de code |
| Convention emergente | `docs/conventions/[backend\|frontend].md` | Formaliser le pattern observe |
| Divergence cross-module | `docs/conventions/[backend\|frontend].md` | Unifier + documenter |
| Heuristique brainstorm | `jeyant-brainstorm/exploration-rules.md` | Ajouter la regle |
| Regle de decoupage | `jeyant-plan/micro-unit-template.md` | Enrichir les guidelines |
| Piege de preuve | `jeyant-preuves/proof-types.md` | Ajouter le piege a eviter |
| Amelioration du pilote | `jeyant/error-protocol.md` ou ressource appropriee | Modifier |
| Nouvelle regle de garde | `jeyant-dev-guard/[ressource].md` | Ajouter |

## regle

Toujours modifier le fichier source, pas creer un nouveau fichier. Les capitalisations enrichissent les ressources existantes.
