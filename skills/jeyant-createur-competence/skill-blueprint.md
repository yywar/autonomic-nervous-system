# blueprint d’une skill

## pièces minimales

- `SKILL.md`
- `workflow.md`
- `output-contract.md`
- `self-review.md`
- `adversarial-review.md`
- `examples/` ou `templates/`

## contenu attendu de `SKILL.md`

- rôle
- mission
- quand utiliser la skill
- quand ne pas l’utiliser
- protocole d’exécution
- sortie minimale attendue
- références aux fichiers annexes

## description

La description doit :
- décrire ce que fait la skill
- indiquer quand l’utiliser
- contenir des mots que l’utilisateur dirait naturellement
- éviter les termes vagues

## règles de robustesse

Une skill robuste :
- explicite ses limites
- sait différencier preuve et supposition
- a une vraie logique de sortie
- ne valide pas trop tôt
- possède une auto-revue et une revue adverse

## règles de proactivité

Une skill proactive :
- complète utilement
- anticipe le besoin adjacent probable
- n’ajoute pas de matière décorative
- reste proportionnée à l’enjeu
