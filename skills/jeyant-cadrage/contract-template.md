# template de contrat de besoin

```markdown
## Contrat de besoin

### Objectif reel
[Quel probleme concret doit etre resolu — pas la demande brute, le besoin sous-jacent]

### Perimetre
[Ce qui est inclus, precisement — modules, fichiers, fonctionnalites]

### Hors perimetre
[Ce qu'on n'est pas cense couvrir — explicite pour eviter les derives]

### Hypotheses
[Prises faute d'information complete — chacune pourrait invalider la solution si fausse]

### Criteres d'acceptation
[Ce qui doit etre vrai a la fin — formulé de maniere testable]
- [ ] critere 1
- [ ] critere 2

### Impact radius
[Modules touches, interfaces modifiees, effets de bord possibles]
- Modules : [liste]
- Contrats/types modifies : [liste ou "aucun"]
- Effets de bord : [liste ou "aucun identifie"]
```
